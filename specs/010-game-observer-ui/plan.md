# Implementation Plan: Game Observer Web UI

**Branch**: `010-game-observer-ui` | **Date**: 2026-03-21 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/010-game-observer-ui/spec.md`

## Summary

Build a read-only web UI for observing live AI vs AI MTG games. The frontend is a React SPA that polls the existing engine REST API, showing an animated game board with card zone transitions, a game list for multi-game selection, and a play-by-play action log. Backend changes are minimal — one new endpoint to list active games and serving the built frontend as static files from FastAPI.

## Technical Context

**Language/Version**: Python 3.11 (backend, matches existing) + TypeScript 5.x (frontend)
**Primary Dependencies**:
- Backend: FastAPI (existing), Pydantic v2 (existing)
- Frontend: React 18, Vite 5 (build tool), TypeScript
**Storage**: N/A (reads from existing in-memory game state)
**Testing**: pytest (backend, existing) + Vitest (frontend unit tests)
**Target Platform**: Modern desktop browsers (Chrome, Firefox, Safari, Edge)
**Project Type**: Web application (SPA frontend + existing REST API backend)
**Performance Goals**: Board state updates within 2s of engine action; animations 300–600ms
**Constraints**: Read-only observer; no WebSocket (polling only); no card artwork; desktop only
**Scale/Scope**: 1–5 concurrent games observed; single local user

**Structure Decision**: Hybrid — existing Python backend with a new `frontend/` SPA directory. The frontend builds to `frontend/dist/` which FastAPI serves via `StaticFiles`. This keeps the frontend isolated from the engine code while allowing a single `uvicorn` process to serve both the API and UI.
