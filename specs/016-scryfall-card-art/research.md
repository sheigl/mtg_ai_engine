# Research: Scryfall Card Art Integration

**Date**: 2026-03-24
**Feature**: [Scryfall Card Art Integration](spec.md)

---

## Decision 1: Image URL Construction — Frontend vs Backend

**Decision**: Construct Scryfall CDN image URLs directly in the frontend using `scryfall_id`. No new backend endpoints required.

**Rationale**: `Card.scryfall_id` is already populated by `ScryfallClient._build_card` and serialized through the existing game API responses. Scryfall publishes a stable, documented CDN URL pattern:

```
Front face: https://cards.scryfall.io/{size}/front/{id[0]}/{id[1]}/{scryfall_id}.jpg
Back face:  https://cards.scryfall.io/{size}/back/{id[0]}/{id[1]}/{scryfall_id}.jpg
```

Where `id[0]` and `id[1]` are the first two characters of the UUID (e.g., `scryfall_id = "abc123..."` → path `/a/b/abc123....jpg`).

**Alternatives rejected**:
- Backend proxy — unnecessary latency/bandwidth; Scryfall ToS permits direct CDN hotlinking for non-commercial apps.
- Expose `image_uris` from cached Scryfall JSON via a new endpoint — adds backend surface area without benefit since the URL is deterministic.

---

## Decision 2: Image Size

**Decision**: Use `small` (146×204px) for in-game card rendering. Use `normal` (488×680px) for hover zoom (if implemented in the future).

**Rationale**: Current card CSS renders at ~80×112px. The `small` format is sufficient quality at that size and loads ~4× faster than `normal`. The Scryfall CDN is fast, but in a 40-card game, 40 simultaneous `normal` fetches would be noticeable.

**Sizes available**: `small` (146×204), `normal` (488×680), `large` (672×936), `png` (745×1040), `art_crop`, `border_crop`.

---

## Decision 3: DFC / MDFC / Transform Cards

**Decision**: Both faces of a DFC/MDFC share the parent card's `scryfall_id`. Use `/front/` for the active face and `/back/` when the permanent is face-down or transformed. The existing `Card.faces` list exists but `CardFace` has no separate `scryfall_id` — the shared parent ID works for both CDN paths.

**Verified**: Scryfall CDN serves `/back/...` for the back face of all transform/MDFC cards using the same parent UUID.

---

## Decision 4: Token Cards

**Decision**: Tokens generated by the engine may not have a real Scryfall ID (`scryfall_id = null`). When `scryfall_id` is absent, fall back to the existing text-based card display (no image rendered). This preserves full functionality for tokens without special-casing.

---

## Decision 5: Browser Caching

**Decision**: Rely entirely on the browser's HTTP cache. No React state, localStorage, or service worker needed for this feature.

**Rationale**: Scryfall CDN sets `Cache-Control: max-age=604800` (7 days) on images. Once a card image is fetched, the browser serves it from disk cache for all subsequent renders in the same and future sessions. For a local development tool, this is fully sufficient.

---

## Decision 6: Card Display Overlay Design

**Decision**: Overlay mode — the card art image fills the card background, with a semi-transparent bottom bar showing card name and P/T stats. Top-right shows mana cost badge. This preserves gameplay information while showing art.

**CSS approach**: `background-image` on `.card` with `background-size: cover`, `background-position: center top`. Bottom overlay uses `rgba` background for legibility.

**Alternatives rejected**:
- `<img>` element inside card layout — harder to achieve full bleed behind stat overlays; CSS background is simpler.
- Art-only, tooltip for stats — removes P/T from battlefield view which is critical during combat.

---

## Decision 7: Face-Down Permanents

**Decision**: Show the official MTG card back image for face-down permanents (`permanent.is_face_down`).

**Card back URL**: `https://cards.scryfall.io/normal/back/0/0/0aeebaf5-8c7d-4636-9e82-8c27447861f7.jpg`

This is Scryfall's official card back placeholder image, stable and documented.

---

## Decision 8: Scryfall Terms of Service

**Decision**: Display images via CDN hotlink; add Scryfall attribution to the app footer.

**Scryfall image policy** (from their docs): Free hotlinking for non-commercial fan content is permitted. Required attribution: card images © Wizards of the Coast, and a "Powered by Scryfall" note.
